Here’s a clear, practical rundown of what `final_scripts/run_batch_pipeline.py` does, what it needs, and how to run it.

**What it does (high level)**
1. Finds COBOL programs in `--cobol-dir`.
2. For each program, it:
   - Enriches PDC JSON, builds controlflow, extracts dataflow, filters used variables, extracts business rules, builds RAG docs.
   - Generates MAPA‑derived architecture/program summary artifacts (from MAPA `result.txt` files).
3. Writes everything under `--out-root` in per‑program folders.

**Required inputs (CLI args)**
- `--cobol-dir`  
  Folder containing COBOL source files (`.CBL`, `.cbl`, `.COB`, `.cob`).

- `--copy-dir`  
  Folder containing COPYBOOK `.cpy` files.

- `--pdc-json-dir`  
  Folder containing per‑program PDC JSONs, or a single JSON file.

- `--out-root`  
  Output root folder (created if missing).

- MAPA input (choose one mode):
  - Legacy single file: `--mapa-result path\to\result.txt`
  - New per‑file mode: `--mapa-batch-dir` and `--mapa-cics-dir`  
    Each dir contains files like `PDCBVC_result.txt`  
    Optional: `--mapa-result-pattern "{program}_result.txt"`

**Optional inputs**
- `--script3` Path to `script3.py` (default: `script3.py` in working dir)
- `--pdc-json-pattern` Default `{stem}.json` (supports `{stem}` and `{program}`)
- `--use-program-id` Use COBOL `PROGRAM-ID` instead of filename stem

**How it works (flow)**
1. Validates inputs (`cobol`, `copybooks`, `pdc json`).
2. Collects COBOL files and determines program name (filename stem or `PROGRAM-ID`).
3. For each program:
   - Runs PDC enrichment + CFG + dataflow + rules + RAG generation.
   - Runs MAPA pipeline:
     - Legacy mode: uses the single `--mapa-result` file once.
     - Per‑file mode: finds the program’s MAPA result in `batch` and/or `cics` dirs, runs `script3.py` separately for each, then generates MAPA‑derived artifacts with a unique suffix.

---

**Outputs (under `--out-root`)**
Per program: `programs/<PROGRAM>/`
- `inputs/`
  - `pdc_typed.json`
  - `pdc_enriched.json`
  - `pdc_dataflow.json`
  - `pdc_var_index_used.json`
  - `pdc_rules.json`
  - `rag_documents.json`
- `artifacts/`
  - `controlflow.cfg.json`
  - `program.summary*.json`
  - `architecture.copybooks*.json`
  - `architecture.calls*.json`
  - `architecture.call*/` (dir)
  - `architecture.sqlinclude*/` (dir)
  - `architecture.db2_table*/` (dir)
  - `dataflow.used_variables.json`
  - `dataflow.variable/` (dir)
  - `business_rule/` (dir)
  - `ui.cics.navigation.json`

If using per‑file MAPA mode, MAPA‑derived outputs have a suffix like:
- `program.summary.batch.PDCBVC_result.json`
- `program.summary.cics.PDCBVC_result.json`

---

**Command to run (per‑file MAPA mode)**
```powershell
python final_scripts/run_batch_pipeline.py ^
  --cobol-dir data\cobol ^
  --copy-dir data\copybooks ^
  --pdc-json-dir data\pdc ^
  --mapa-batch-dir data\mapa\batch ^
  --mapa-cics-dir data\mapa\cics ^
  --mapa-result-pattern "{program}_result.txt" ^
  --out-root out
```

**Command to run (legacy single MAPA file)**
```powershell
python final_scripts/run_batch_pipeline.py ^

  --copy-dir data\copybooks ^
  --pdc-json-dir data\pdc ^
  --mapa-result data\mapa\result.txt ^
  --out-root out
```
